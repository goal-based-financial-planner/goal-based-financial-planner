# Implementation Plan: Export Financial Plan as PDF

**Branch**: `010-export-plan-pdf` | **Date**: 2026-04-12 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/010-export-plan-pdf/spec.md`

---

## Summary

Add an "Export" dropdown button to the Planner header that offers "Download PDF" (produces a named, downloaded file via jsPDF + html2canvas) and "Print" (opens the browser print dialog via `window.print()`). A dedicated `PrintableReport` component — hidden from the interactive UI, visible only during print/export — renders all plan sections (goals, investment suggestions, allocation data, investment history, progress summaries) in a simplified, single-column layout. Export logic is encapsulated in a `usePdfExport` hook. No new persistent data entities; the export is a point-in-time snapshot of the in-memory `PlannerData`.

### Component interfaces

#### `ExportButton` props

```typescript
type ExportButtonProps = {
  onDownloadPdf: () => Promise<void>;
  onPrint: () => void;
  isExporting: boolean;
};
```

- `onDownloadPdf`: async handler that triggers PDF generation via `usePdfExport`
- `onPrint`: synchronous handler that calls `window.print()`
- `isExporting`: when true, button is disabled and shows a CircularProgress spinner

#### `PrintableReport` props

```typescript
type PrintableReportProps = {
  plannerData: PlannerData;
  selectedDate: string;
  investmentBreakdownBasedOnTermType: InvestmentBreakdownBasedOnTermType[];
  termTypeWiseProgressData: TermTypeWiseProgressData[];
  printRef: React.RefObject<HTMLDivElement>;
};
```

#### `usePdfExport` return shape

```typescript
type UsePdfExportReturn = {
  isExporting: boolean;
  error: string | null;
  downloadPdf: (ref: React.RefObject<HTMLDivElement>) => Promise<void>;
  print: () => void;
};
```

### `PrintableReport` sections (rendered in order)

1. **Report Header** — App title ("Goal Based Financial Planner"), export date, "as of" month/year
2. **Financial Goals** — Table: Goal Name | Target Amount | Target Date | Term Type | Progress %
3. **Investment Allocation Plan** — Per term (Short/Medium/Long): table of instrument name + monthly amount + expected return %
4. **Investment History** — Table: Date | Investment Type | Amount logged (from `investmentLogs`)
5. **Term-wise Progress** — Table: Term | Goals | Target Amount | Progress %
6. **Empty state** — When no goals exist: single paragraph "No financial goals defined yet."

Each section except the header is preceded by a `<div className="print-page-break">` to ensure clean pagination.

### `@media print` CSS rules

Added to `src/index.css`:

```css
@media print {
  /* Hide the entire interactive app */
  body > #root {
    display: none;
  }
  /* Show only the print report */
  .printable-report {
    display: block !important;
  }
  /* Page setup */
  @page {
    margin: 1cm;
    size: A4 portrait;
  }
  /* Page breaks before each major section */
  .print-page-break {
    page-break-before: always;
  }
}

/* Hide from screen by default */
.printable-report {
  display: none;
}
```

### `usePdfExport` implementation approach

```
downloadPdf(ref):
  1. Set isExporting = true, error = null
  2. Dynamically import html2canvas and jsPDF (code-split, avoids adding to main bundle)
  3. Call html2canvas(ref.current, { scale: 2, useCORS: true, logging: false })
  4. Create jsPDF instance (A4, portrait)
  5. Calculate image dimensions to fit A4 width, add pages as needed
  6. Save as "financial-plan-{dayjs().format('YYYY-MM-DD')}.pdf"
  7. Set isExporting = false
  8. On any error: set error = human-readable message, set isExporting = false

print():
  1. Call window.print()
  (CSS @media print handles showing/hiding content)
```

## Implementation Notes

### Dynamic import for PDF libraries

To avoid bloating the main JS bundle, `jsPDF` and `html2canvas` are imported dynamically inside `downloadPdf`:

```typescript
const [{ default: html2canvas }, { jsPDF }] = await Promise.all([
  import('html2canvas'),
  import('jspdf'),
]);
```

This means the ~100KB brotli cost is only paid when the user actually triggers "Download PDF", not on initial page load.

### Multi-page PDF handling

`html2canvas` produces a single tall canvas. `jsPDF` requires explicit page addition:

```
canvasHeight / canvasWidth * pdfWidth = pdfPageHeight (for first page)
If canvas is taller than one A4 page:
  add pages, translate canvas Y position accordingly
```

A utility function `addCanvasToMultiPagePdf(canvas, doc)` handles this inside `usePdfExport`.

### PrintableReport rendering during html2canvas capture

`html2canvas` captures the current DOM state. Since `PrintableReport` is always mounted (just CSS-hidden), `html2canvas` can target it directly via the ref without needing to temporarily show it on screen. The `scale: 2` option doubles resolution for retina-quality output.

### Error handling

Errors from `html2canvas` or `jsPDF` are caught and surfaced as a string error message. The `ExportButton` renders this message as a MUI `Alert` component (severity "error") directly below the button when `error` is non-null. The error clears on the next export attempt.

### `Planner` prop change

A new `printRef` prop is added to `Planner`:

```typescript
type PlannerProps = {
  plannerData: PlannerData;
  dispatch: Dispatch<PlannerDataAction>;
  headerRight?: React.ReactNode;
  printRef?: React.RefObject<HTMLDivElement>;  // new, optional
};
```

`printRef` is optional so existing test renders of `Planner` don't break.
